# Code Style: Simple Procedural
Write linear, procedural code without functions or abstractions. Prioritize immediate readability over DRY principles:
- No helper functions, wrappers, or utilities
- Inline all logic in execution order (top to bottom)
- Repeat code blocks rather than abstracting them
- Use descriptive variable names and comments to explain sections

If code needs to happen multiple times, copy the block. Code should be understandable without jumping between definitions.
